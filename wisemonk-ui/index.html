Wisemonk UI

A portable, framework-free component library extracted from the Wisemonk portal. Drop wisemonk-ui.css + wisemonk-ui.js + the icon sprite into any project. Everything is namespaced (wm- classes, --wm- tokens) so nothing collides.

App shell — nav & header

The exact navigation sidebar and header from the live freelancer portal — copied verbatim (classes .sidebar / .nav-link / .nav-section / .header / .user-chip). Includes the collapsible group (Time → Leaves / Attendance) and the user chip. Source: app-shell.html.

Click Time to collapse the group. Open app-shell.html full-page, or resize below 900px to see the sidebar hide (portal behavior).

Dev notes — design annotations

Add data-wm-note="…" to any element to leave a note for developers. Click the floating Dev notes button (bottom-right) to reveal numbered pins, then hover any element to read its note — no panel needed. (Click a pin for the full list.) Hidden from end-users by default; open the mode with the button, the Alt N shortcut, or a ?notes URL parameter. The elements below are annotated — toggle Dev notes and hover them.

Selectors & dates

Exact ports of the portal's form controls — the custom dropdown keeps the native <select> as its trigger and overlays a styled menu; the date picker turns a native <input type="date"> into a calendar popover. Both live inside a .wm-field-float and submit with the form. The multi-select reuses the same menu markup, ticking several options. Just give the select class="wm-select" (add multiple for multi-select).

Onboarding form

The portal's signature floating-label fields (.wm-field-float) combined with custom selectors, the date picker and inline validation. The label sits centered as the placeholder, then lifts on focus / once filled. Add data-wm-validate to the <form>; required fields show inline errors on submit and clear as the user fixes them (wm:valid / wm:invalid).
